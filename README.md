aquazone 風のターミナル熱帯魚育成ツール。Rust + crossterm による自前ハーフブロック描画で、ブルー系の水槽に熱帯魚が泳ぎます。餌をやって育て、成長・繁殖させて眺めるアクアリウムです。

## 特徴

- crossterm による自前レンダリング(ratatui 非依存)。ハーフブロック文字 `▀` で縦2倍解像度のカラー描画、前フレームとの差分のみ書き込みちらつきを抑制
- 魚は f64 座標で保持しなめらかに遊泳(ランダムウォーク+慣性、壁で緩やかに反射、同種で軽く群れる、腹ぺこだと通常の遊泳よりはっきり優先して餌に一直線に向かう)。低頻度・ランダムなタイミングで一瞬だけ速く動く「瞬発ダッシュ」も入る
- 7種類の魚(ネオン=青系 / 金魚=オレンジ / グッピー=白+差し色 / エンゼルフィッシュ=縦長で優雅 / ベタ=派手なヒレ / ピラニア=灰青の大型捕食者 / タコ=もう1種類の捕食者)。ドットマトリクスは病気のまだら模様・ヒレの形がはっきり見えるレベルまで拡大している。稚魚と成魚で見た目が変わり、ピラニア・タコも含めて空腹度・成長・病気・死亡・寿命の育成ロジックに参加する(可動範囲もスプライトの拡大サイズに応じて広がるので、大きく育った魚が水底に埋まって見えたりしない)
- ピラニアは捕食者: 空腹時に他の魚(タコも対象)を検知して追跡(追跡中は通常種より速い)して襲い、捕食されると強化した血飛沫演出(複数色の破片パーティクル+同心円状に広がる血の滲み)とともにその場で消える。捕食するたびに段階的に大きくなる。**ピラニアの入手経路は`S`キーのみ**(初期配置・`+`キーのランダム追加・産卵→孵化のいずれにもピラニアは含まれない)
- タコは水底の「タコつぼ」に隠れていて普段は姿が見えない、もう1種類の捕食者。低頻度でつぼから出てきて泳ぎ、しばらくしたら戻る。ピラニアに追われると墨を吐いて逃げる(墨の範囲にいる捕食者は獲物を検知できなくなる)。**タコの入手経路も`O`キーのみ**(初期状態ではタコつぼだけが空の状態で配置され、タコ自体は1匹もいない。`O`キーで追加すると空いているタコつぼに入居する)
- 通常の魚は、ピラニアの空腹状態に関わらず近くにピラニアがいると常に回り込むように(ジグザグに)逃げる。ガラスを叩かれた驚き逃げも同様に機敏でジグザグ、いずれも逃走のたびに空腹度を消費する。藻・水草の近くにいると視覚的に「隠れている」表現になる
- 育成要素: 空腹度(見た目・オーバーレイに反映)・成長(稚魚→成魚、その後も満腹維持でさらに段階的にサイズアップ)・産卵→孵化(ピラニア・タコを除く5種)・老齢での確定産卵と寿命死(全種共通)・病気と治療・死亡(猶予は長め、死亡演出つき)
- 水底・中層に藻・水草(ゆらゆら揺れる装飾)・岩・タコつぼ(タコの巣)を配置。魚がすっぽり身を隠せるサイズがあり、藻・岩に隠れている魚は実際にピラニア・タコから捕食されなくなる(見た目だけでなく機能として無敵)。育成ロジック自体(空腹度等)には参加しない静的なオブジェクト
- カーソル(照準)で餌・薬の投下位置を指定でき、投下位置には一瞬光る演出が出る。ガラスを叩いて(`t`)近くの魚を驚かせることもできる(叩きすぎるとストレスで病気になりやすくなる)
- 自動モード(`a`): ONにすると、腹ペコ/病気の魚がいるときに自動で餌やり・投薬を行い、低頻度でランダムにガラスも叩く(いずれも一定間隔のクールダウンあり)
- 設定画面(`,`): 効果音・オーバーレイ・自動モードのON/OFFを一覧して切り替えられる
- 各魚の直上にステータスオーバーレイ(腹ペコ/病気フラグ・生命残りゲージ)を点滅表示(ON/OFF切替可)
- カニは観賞用の背景生物として水底を歩く(育成ロジック対象外)。水底に沈んだ食べ残しの餌・薬を片付ける掃除役も兼ねる(1tickで1つまでのゆっくりした掃除)
- 個体数上限は端末サイズに応じて最大100匹まで(魚のドット絵拡大に合わせて収容密度は下げてある)。`+`キーでの直接追加は25匹まで、それ以上は産卵→孵化でのみ到達する
- シミュレーション速度は0.25倍〜16倍まで7段階(`[` `]`)
- 効果音(周波数スイープ+速い減衰で合成したビープ音、ON/OFF切替可)。オーディオデバイスが無い環境でもクラッシュしない
- 起動時にタイトルロゴのスプラッシュ画面を表示(`?` のヘルプ画面にも同じロゴ)。ヘルプ画面には全7種の図鑑(実際のドット絵スプライト付き)も表示する(端末が狭い場合はテキストのみにフォールバック)
- 一時停止・速度変更・グレートリセットで観察やデバッグがしやすい
- 終了時に状態を保存し、次回起動で続きから再開

## ビルドと起動

```sh
cargo build --release
./target/release/aquaterm
```

起動時に毎回タイトルロゴのスプラッシュ画面が出ます(何かキーを押すか少し待つと消えます)。初回起動時はさらにヘルプが自動表示されます(何かキーを押すと水槽へ)。

### 音無しでビルドする(`sound` feature)

効果音(`rodio`)は既定でON(`sound` feature)ですが、`rodio`の依存(`alsa-sys`)がLinux向けクロスコンパイル環境等でビルドに失敗する場合、音無しでビルドできます:

```sh
cargo build --release --no-default-features
```

`sound` featureを外すと`rodio`自体をビルドに含めなくなる(依存を丸ごと切り離す)ため、ALSA等のオーディオ関連ライブラリが一切無い環境でも問題なくビルドできます。呼び出し側のコード(`sim.rs`/`main.rs`)は変更不要で、`SoundEngine`が何もしないダミー実装に切り替わるだけです。

### Linux での動作について

開発は主にmacOS上で行っています。音無し(`--no-default-features`)版は、Linux実機上での動作を確認済みです:

- macOSから`x86_64-unknown-linux-musl`ターゲットへのクロスコンパイル(音無し)で静的リンクのLinux用ELFバイナリを生成:
  ```sh
  export CC_x86_64_unknown_linux_musl=x86_64-linux-musl-gcc
  export CARGO_TARGET_X86_64_UNKNOWN_LINUX_MUSL_LINKER=x86_64-linux-musl-gcc
  cargo build --release --target x86_64-unknown-linux-musl --no-default-features
  ```
- 生成したバイナリをUbuntu 24.04(x86_64)の実機に転送して起動し、スプラッシュ・ヘルプ画面が正しく表示され、クラッシュしないことを確認済み
- 音あり(既定feature)でのクロスコンパイルは、クロスコンパイル環境にALSAのsysrootが無いため`alsa-sys`のビルドに失敗する(pkg-config がALSAを見つけられない)。音ありでLinux上でビルドしたい場合は、そのLinuxマシン上で直接`cargo build --release`する(下記のALSA開発パッケージが必要)

音あり(既定feature)でLinux上でビルドする場合、`cpal`が既定でALSAバックエンドを使うため、**ビルド時にALSAの開発用ヘッダが必要**です。Debian/Ubuntu系では事前に以下を入れてください:
```sh
sudo apt install libasound2-dev pkg-config
```
(Fedora/RHEL系では `alsa-lib-devel` 相当のパッケージ)

その他、コードレベルで確認した限りでは以下の理由からLinux上でも問題なく動くと考えられます(未検証だが、依存クレートの対応状況からおそらく動くと考えられる、というレベルの記載です):

- 端末描画(`crossterm`)はUnix系(Linux/macOS共通)で `termios` ベースの実装を使うクレートで、Linuxサーバーでの利用実績も多く、特別な対応は不要と考えられる
- オーディオ出力デバイスが無いヘッドレスサーバーでも、`SoundEngine::new()`はデバイス初期化失敗を握りつぶしてSEなしで動作を継続する実装になっている(コード上は`Err(_) => SoundEngine { sink: None }`)ため、音声デバイスが無い環境でも動作するはず
- 状態保存先(`~/.config/aquaterm/state.json`)は`$HOME`環境変数を直接参照する自前実装で、OS固有のAPI(macOSの`~/Library/Application Support`等)には依存していない
- コード内にmacOS固有のAPI呼び出し(`cfg(target_os = "macos")`等)は見当たらなかった

## キー操作

| キー      | 動作                                                   |
|-----------|--------------------------------------------------------|
| 矢印キー  | カーソル(照準)を移動                                  |
| `f`       | カーソル位置から餌やり(3〜5粒投下、投下位置が一瞬光る) |
| `m`       | カーソル位置から薬を投げる(病気の魚が触れると治癒)     |
| `t`       | ガラスを叩く(近くの魚が驚いて逃げる。叩きすぎ注意)     |
| `T`       | トントン(近くの魚が興味を持ってカーソル位置に寄ってくる。`t`の逆効果) |
| `p`       | 一時停止 / 再開                                        |
| `[` / `]` | シミュレーション速度を下げる / 上げる                   |
| `R`       | 水槽グレートリセット(確認あり)                        |
| `v`       | ステータスオーバーレイ(腹ペコ/病気/生命ゲージ)の表示切替 |
| `s`       | 効果音(SE)のON/OFF                                    |
| `a`       | 自動モード(自動餌やり/投薬/ガラス叩き)のON/OFF        |
| `A`       | 自動魚補充のON/OFF(通常魚が減ったら自動で1匹追加。`a`とは別トグル) |
| `+` / `-` | 魚を1匹追加(通常5種からランダム) / 間引く(デバッグ用) |
| `S`       | ピラニアを1匹確実に追加(ピラニアを増やせる唯一の方法)         |
| `O`       | タコを1匹確実に追加(タコを増やせる唯一の方法)         |
| `D`       | タコつぼを再配置(既存を消して新しい位置に生成し直す)   |
| `P`       | 藻・水草を再配置(既存を消して新しい位置に生成し直す)   |
| `,`       | 設定画面を開く(効果音/オーバーレイ/自動モードを一覧切替) |
| `?`       | ヘルプの開閉(全7種の図鑑つき)                         |
| `q` / Esc | 終了(状態を保存)                                     |

設定画面(`,`)を開いている間は、上下キーで項目選択・Enter/スペースでトグル・Escで画面を閉じます(水槽の操作には伝わりません)。

## 育成のしくみ

- **空腹度**: 時間で減少し、餌を食べると回復する。腹ぺこになると通常の遊泳(ランダムウォーク・群れ)よりはっきり優先して最寄りの餌へ一直線に向かう。満腹/普通/腹ぺこの3段階で泳ぐ速さと色も変わる(腹ぺこは速く泳ぎ色が薄暗くなる)
- **ステータスオーバーレイ**: 各魚の直上に、腹ペコ/病気フラグと「生命残りゲージ」(空腹度と病気の深刻度から算出、緑〜黄〜赤のグラデーション)を点滅表示する。`v` キーでON/OFFを切り替えられる
- **成長**: 満腹を維持し続けると稚魚が成魚に育ち、サイズが大きくなる。成魚になった後も満腹維持を続けると、さらに段階的にサイズアップする(全種共通・上限あり。大きくなるほど泳ぐ速度がわずかに遅くなる)
- **サイズと機敏さの連動**: 「大きくなるほど遅くなる」と対になる形で、稚魚や成長段階が低い(体格の小さい)個体ほど、特別なトリガー(空腹・逃走等)が無い通常の遊泳(ランダムウォーク+慣性)でもキビキビと素早く・方向転換も頻繁になる。サイズが大きくなるほどその逆でゆったりする(同じサイズの軸から滑らかに連動)
- **繁殖(産卵→孵化)**: 満腹の成魚は水底付近に卵を産む(頻度は低め・数十分に1回程度を目安に調整済み)。産卵の瞬間、卵の位置に数秒間キラキラ光るフラッシュ演出が出る。卵は一定時間で孵化して稚魚になる(個体数上限を超える分は孵化しない)。**ピラニア・タコは産卵しない**(ピラニアが増える唯一の方法は`S`キー、タコが増える唯一の方法は`O`キー)
- **寿命・世代交代**: 各個体は生まれてからの経過時間(年齢)を記録し、老齢に達すると満腹状態などの条件を問わず確定で1回だけ産卵する(次世代を残す最後のチャンス。ピラニア・タコは対象外)。さらに年を取ると老衰で寿命死する(既存の死亡演出を使う)。対象は全種共通(ピラニア・タコも含む。無限に居座り続けない)
- **病気**: 腹ぺこが続いたり過密になると発症する(頻度は低め・最高速度で遊んでいて水槽全体で10分に1回程度を目安に調整済み)。まだらにくすんだ色になり泳ぎが鈍り、成長・繁殖が止まる。`m` の薬で治療できる(健康な魚には無害)。ガラスを叩きすぎると周辺の魚が一時的に発症しやすくなる
- **死亡**: 空腹度0または病気の放置が長時間(それぞれ猶予10分以上)続くと力尽きる。死んだ魚は仰向けのスプライトに変わってゆっくり水面近くまで浮上し、しばらく(15秒程度)水槽に留まったのち消える。餌や薬を与えて回復すれば死亡は避けられる
- **可動範囲とサイズ拡大**: 成長段階(render_scale)で拡大されたスプライトの実サイズに応じて、壁際・水底の可動範囲マージンも動的に広がる。大きく育った魚ほど中心座標が壁・水底に近づきすぎないようにすることで、拡大後の見た目が水底に埋まったり壁に張り付いたりせず、水槽全体を自由に泳ぎ回れるようにしている
- **ピラニアの捕食**: ピラニアは空腹時、他の魚(ピラニア以外・タコも対象)を検知して追跡し、触れると捕食する。追跡(捕食モード)中はピラニアの速度・加速度・慣性(ブレーキの弱さ)がまとめて強化され、通常種よりはっきり速く追いつく体感になる(巡回中は特別早くしない)。頻繁に狩る体感になるよう、空腹度の閾値・クールダウン・検知範囲も強化してある。壁際・角に追い詰められた獲物も、追跡中は壁際のマージンが基本値に戻るため確実に捕食できる(捕食者・被食者でスプライトサイズが違っても、壁際の可動範囲の差で埋まらない隙間ができないようにしている)。捕食された魚は強化した血飛沫演出(内臓を思わせる複数色の破片パーティクル+同心円状に広がって数秒間まっかに見えてからゆっくりフェードアウトする血の滲み)とともに即座に消え、ピラニアの空腹度は大きく回復し、ピラニア自身は捕食するたびに段階的に大きくなる(上限あり)
- **ピラニアのカニバリズム**: ピラニア同士は基本的に捕食対象にならないが、十分サイズ差(成長段階+捕食成長段階)がある場合に限り、大きいピラニアが小さいピラニアを捕食できる(近いサイズ同士は対象外のまま)
- **ピラニアへの警戒・逃走**: 通常の魚(タコも含む)は、ピラニアが空腹かどうかに関わらず近くにピラニアがいると常に逃げる(「みんなピラニアが嫌い」)。逃走は真っ直ぐ離れるだけでなく回り込むようにジグザグに切り返し、通常の遊泳よりはっきり優先される機敏な動きになる。ガラスを叩かれた驚き逃げも同様にジグザグかつ機敏。いずれの逃走も、始まった瞬間に空腹度を消費する(逃げるのにエネルギーを使う想定。危険が続く間の再課金はない)
- **トントン(`T`)**: `t`(コンコン)の逆効果。カーソル位置を軽くノックすると、近くの魚が驚いて逃げる代わりに興味を持ってカーソル位置へ穏やかに近づいてくる。見た目(波紋の色)・効果音も`t`とは差別化した柔らかいものにしている
- **ランダムな瞬発ダッシュ**: ピラニア・餌などのトリガーが無い通常の遊泳中でも、低頻度・ランダムなタイミングで一瞬だけ速く動く(数十秒に1回あるかないか程度)
- **ピラニアの入手経路**: `S`キーのみ。初期配置・グレートリセット・`+`キーのランダム追加・産卵→孵化には一切含まれない(狙って入れたときだけ水槽に現れる)
- **タコ**: もう1種類の捕食者。初期状態ではタコつぼだけが空で配置され、タコ自体は1匹もいない(デフォルトで自動的に増えることはない)。通常は水底のタコつぼに隠れていて姿が見えず、低頻度でつぼから出てきて泳ぎ、しばらく(20〜40秒程度)したら自分でつぼへ戻ってまた隠れる(隠れる時間は15〜40秒程度)。タコ自身の捕食も低頻度・空腹時のみ・クールダウンありで、ピラニアと同様に他の魚(ピラニアは対象外)を狩れる。ピラニアに追われると(近くに捕食モードのピラニアがいると)、逃走に加えて墨を吐く。タコを増やせる方法は`O`キーのみ(空いているタコつぼがあればそこに入居し、無ければ新しいタコつぼも1つ生成される)
- **墨**: タコが吐く、ほぼ純黒の墨が勢いよく同心円状に大きく拡散し、数秒間残ってからゆっくり薄れて消える(血の滲みより広め・速い拡散・視界をしっかり覆う濃さにしてある)。墨が広がっている間、その範囲にいる捕食者(ピラニア等)は獲物を検知できなくなる(視界が悪くなる演出)。加えて、墨を吐いた瞬間のタコには緊急ダッシュ(速度・逃走ベクトルの一時的な強化)がかかり、直後の一定時間は捕食判定からも除外されるため、高確率でその場を逃げ切れる
- **スター(無敵アイテム)**: 低頻度・ランダムな位置・タイミングで水槽内にキラキラ光るスターが出現する(数分〜十数分に1回程度)。魚(通常種・ピラニア・タコいずれでも)が触れると取得し、10〜20秒ほど無敵状態になる(光る/点滅する見た目に変わる)。無敵中は誰にも捕食されず、逆に触れた他の魚(ピラニア・タコを含む)を種類に関わらず捕食できる(マリオのスターのように、普段は捕食されない側が一時的に捕食者へ逆転するギミック)。無敵時間が終わると通常の育成ロジックに戻る
- **藻・水草・岩・タコつぼ**: 水底・中層に配置する静的な装飾オブジェクト(ゆらゆら揺れるアニメーションのみで自身は育成ロジックに参加しない)。魚のドット絵より明らかに大きく、完全に覆い隠せるサイズにしてあり、藻・岩に十分近い(隠れている)魚は見た目が背景に馴染むだけでなく、実際にピラニア・タコから捕食されなくなる。タコつぼはタコの巣として機能し、`D`キーで再配置すると同時に、タコつぼの数も現在のタコの数にきちんと揃えられる(タコより多ければ削除・少なければ追加)
- **カニ・エビ・タツノオトシゴ**: いずれも育成ロジック対象外・捕食対象外(自身も捕食しない)の観賞用背景生物。カニ・エビは水底を左右に歩き、時々立ち止まる(カニは停留した食べ残しの餌・薬を片付ける掃除役も兼ねる。1匹が1tickで片付けられるのはそれぞれ1つまでなので、山になっていても一気には無くならずゆっくり片付く)。タツノオトシゴは藻に絡みつくようにゆっくり漂い、藻の近くからあまり大きく移動しない
- **カメオ生物(ウミガメ・クラゲ・小魚の群れ)**: 低頻度(数分〜十数分に1回程度)で画面の端に出現し、反対側の端までゆっくり通過して消える完全観賞用の生物。育成ロジック・捕食判定のいずれにも参加せず、捕食対象にならず自身も捕食しない(見た目のバリエーションを増やすためだけの存在)
- **昼夜の照明変化**: 実際のシステム時刻に応じて、水槽の照明(水のグラデーション・砂地の明るさ)が自動で変化する。昼間(6:00-18:00)は通常の明るいトーン、夜間(18:00-6:00)は暗く落ち着いたトーンになり、境界の前後1時間でなめらかに切り替わる(パキッと変わらない)。設定画面(`,`)でON/OFFを切り替えられる(OFFにすると常に昼間相当の明るさに固定される)
- **水底の食べ残し**: 餌・薬は水底に着くと停留し(寿命が止まる)、カニに片付けられるか、停留数が上限を超えると古いものから消える。1tickで1匹の魚が消費できる餌・薬は1つまでなので、密集させても一気には無くならない。同じ場所へ大量に投入し続けると、はっきり「山」として認識できるレベルまで盛り上がる
- **個体数上限**: 端末サイズ(論理ピクセル数)に応じて決まる(魚のドット絵拡大に合わせて収容密度を調整している)。`+`/`S`キーでの直接追加は25匹まで。それ以上(25〜端末依存の上限・最大100)は産卵→孵化を経由してのみ到達する(ただしピラニア・タコは産卵しないため、ピラニアの数は`S`キーを押した回数分・タコの数はつぼの数までにしかならない)
- **自動モード(`a`)**: ONの間、腹ペコの魚がいて漂っている餌が少なければ一定間隔(数十秒に1回程度)で自動的に餌やりし、病気の魚がいて漂っている薬が少なければ同様に自動投薬する。加えて低頻度(数分に1回程度)でランダムにガラスを叩く。投下位置・叩く位置はランダム(カーソル位置にはこだわらない)。ステータスバーに自動モードのON/OFF状態を表示する
- **自動魚補充(`A`)**: `a`(自動モード)とは別のトグル。ONの間、通常魚(捕食対象になる種。ピラニア・タコ・カメオ生物は含まない)の生存数が少なくなったら、`+`キー相当(ランダムな通常種を1匹追加)を自動的に行う。頻発しすぎないようクールダウンがあり、個体数上限はそのまま適用される。ステータスバーに自動モードとは区別して表示する
- **設定画面(`,`)**: 効果音・ステータスオーバーレイ・自動モード・昼夜連動の照明変化・自動魚補充のON/OFFを一覧表示し、上下キーで選択・Enter/スペースで切替・Escで閉じる(それぞれ`s`/`v`/`a`/`A`キーと状態を共有する、分かりやすい入口)
- **効果音**: 気泡・餌やり・投薬・発症・治療・空腹・ガラスを叩く・トントン・捕食・墨を吐く瞬間に短いビープ音を鳴らす(`s` でON/OFF、既定はON)。外部音源ファイルは使わず、周波数スイープ+速い減衰の正弦波をその場で合成している
- **時間制御**: `p` で全体を一時停止、`[` `]` で速度を段階変更(空腹度減少・成長・孵化・寿命など時間経過ロジック全体に倍率がかかる)

## 状態の永続化

`~/.config/aquaterm/state.json` に各魚(ピラニア・タコ含む)の種類・成長段階・空腹度・病気状態・年齢・タコの隠れ状態・死亡演出の進行度・座標、卵・餌・薬・カニ・エビ・タツノオトシゴ・スターの状態、経過時間を保存します。`q` 終了時に保存し、起動時に読み込んで再開します(ファイルが無ければ初期状態で開始)。藻・水草・岩・タコつぼは保存対象ですが、血飛沫・血の滲み・墨・カメオ生物など一瞬〜数十秒で消える演出、自動モード/自動魚補充/昼夜連動のON/OFFは保存対象外です(次回起動時は既定状態に戻ります)。

## テスト

```sh
cargo test
```

空腹度の減衰・成長判定(通常成長+成魚後の追加サイズ成長)・サイズと機敏さの連動・繁殖(個体数上限・産卵フラッシュ演出を含む)・寿命/老齢確定産卵・病気の発症/治療・死亡の猶予と死亡演出後の消滅・ピラニアの捕食/追跡速度/成長/カニバリズム・逃走(回り込み・逃走コスト)・トントン(引き寄せ)・壁際・角での確実な捕食・ランダムダッシュ・タコの隠れる/出てくる状態遷移(初期状態では0匹)・タコつぼの数をタコの数に揃える再配置・墨(視界不良・緊急脱出保証)・藻/岩に隠れた魚の被食免除・血飛沫演出・食べ残しの山の盛り上がり・スター(無敵アイテム)・カメオ生物の出現/通過/消滅・エビ/タツノオトシゴの挙動・昼夜の明るさ変化・自動モード・自動魚補充・効果音イベントの発火などの育成ロジックを、154件の単体テストで検証しています。`cargo build --release --no-default-features` で効果音(rodio)自体をビルドから切り離すこともできます(Linux向けクロスコンパイル等でALSAが無い環境向け)。オーディオ出力デバイスが無い環境でも `cargo build` / `cargo test` は問題なく通ります(効果音の初期化失敗は握りつぶされます)。
