# aquaterm 仕様書 (draft v1)

## 概要
aquazone 風の熱帯魚育成ツールをターミナルで動かす。termmap と同じく Rust + crossterm で自前レンダリングし、見た目のテイストを termmap に揃える(黒背景・下部反転ステータスバー・ハーフブロックによる高密度カラー描画)。配色はブルー系。

## 技術スタック
- Rust (edition 2021)
- crossterm: 端末描画・キー入力
- unicode-width: 全角文字幅計算(ステータスバー等)
- serde / serde_json: 状態の保存・読み込み
- ratatui は使わない(termmap と同じく自前 diff 描画。half-block で実質2倍解像度を出すため)

## 画面構成
- 上部: 水槽エリア(画面幅いっぱい、高さ = 端末高さ - 1)
- 下部: ステータスバー(反転表示)。餌の残数・魚の数・経過時間・キー案内を表示
- 起動時オンボーディング: termmap 同様、初回のみ簡易ヘルプを表示(以後出さない)

## カラースキーム(ブルー系)
- 背景(水): 上部が明るいシアン〜中段が青〜下部が濃紺のグラデーション(3〜4段階)
- 水底: 暗めのネイビー+砂の差し色(薄い黄土色を少量)
- 気泡: 白〜水色の点、上に向かってゆっくり移動
- 魚: 種ごとに差し色(オレンジ/イエロー/白など)を水槽の青との対比で目立たせる
- ステータスバー: termmap と同じ反転(明るい背景+黒文字)

## 描画方式
- 内部フレームバッファは「セル高さの2倍」の解像度を持つ論理ピクセルグリッド(上下2ピクセル分を上半分/下半分の色として1文字の半角ブロック `▀` に合成、termmap の half-block レンダラと同じ考え方)
- 前フレームとの diff のみ crossterm に書き込み、ちらつきを抑える
- tick は 30〜60fps 目標(可変、端末/描画コストに応じてフレーム間引き)。魚の座標は f64 で保持し、描画時にセル座標へ変換 → 補間により「ぬるぬる」動く

## 魚の挙動
- 座標・速度ベクトル(f64)を保持し、緩やかなランダムウォーク+慣性で自然に泳ぐ
- 水槽の壁で反射(端で急停止せず、手前で緩やかに向きを変える)
- 同種同士は緩く群れる(近傍平均方向に少し寄せる程度の軽い boid。過剰な実装はしない)
- 空腹度が低い個体は餌の位置に引き寄せられる(通常の遊泳ベクトルに餌方向への吸引ベクトルを加算)

## 魚種(dot-matrix デザイン、複数種)
- MVP では 3 種類程度を用意(例: ネオンテトラ風の小型青系、金魚風のオレンジ、グッピー風の白+差し色)
- 各種でサイズ・泳ぐ速さ・餌への反応速度が少し異なる

## 育成要素
- **空腹度**: 時間経過で減少。餌を食べると回復。0 が一定時間続くと弱り、さらに続くと死んで水槽から消える(通知はステータスバーに一言出す程度)
- **空腹度の見た目反映**: 空腹度を3段階(満腹/普通/腹ぺこ)に分け、魚の挙動と色に反映する
  - 満腹: 泳ぐ速度が遅くゆったり、色は通常色のまま
  - 普通: 標準の速度・色
  - 腹ぺこ: 泳ぐ速度が上がり餌への吸引ベクトルが強まる、色をやや薄く/暗くして「弱っている」印象を出す
- **成長**: 満腹状態が続くと稚魚→成魚へ段階的に成長(見た目サイズが変わる)
- **繁殖(産卵→孵化の2段階)**:
  - 成魚が満腹状態を一定時間維持すると、水底や水草付近に卵(2〜4個、小さいドット)を産む
  - 卵は空腹度を消費せず、孵化までの時間(一定時間)が経過すると孵化して稚魚になる
  - 孵化予定数が水槽の個体数上限を超える分は孵化しない(卵は消える)
- **個体数上限**: 端末サイズに応じた上限を設定し、増えすぎて処理が重くならないようにする

## 餌やり操作
- `f` キー: 画面中央上部から餌粒を複数個(3〜5粒)投下
- 餌はゆっくり沈み、水底 or 一定時間経過で消える
- 魚が餌に近づくと自動で食べる(空腹度回復、餌は消滅)

## 病気・治療
- **発症**: 腹ぺこ状態が一定時間続く、または個体数が上限に近い過密状態のとき、確率で発症。発症すると見た目がまだら/くすんだ色になり、泳ぐ速度が鈍る。成長・繁殖は病気の間停止する
- **放置した場合**: 空腹度0の衰弱と同様、一定時間放置すると弱り、さらに続くと死んで消滅する
- **治療**: `m` キーで薬(餌と別色の粒)を投下。ゆっくり沈み、病気の魚が触れると治癒して通常状態に戻る。健康な魚が触れても何も起きない(誤投下してもマイナス影響なし)
- ステータスバーに「病気: N匹」を表示

## 水槽リセット・時間制御
- **グレートリセット**: `R`(大文字)キーで水槽を初期状態に戻す。誤操作防止のためステータスバーに確認プロンプトを出し、明示的な追加キー入力があった場合のみ実行する。魚を初期構成に戻し、卵・餌・薬・経過時間をすべて消去する
- **一時停止/再開**: `p` キーでシミュレーション全体(魚の動き・空腹度減少・成長・繁殖判定など)を停止/再開する
- **速度倍率**: `[` / `]` キーでシミュレーション速度を段階的に変更する(例: 停止相当の極小速度 〜 通常速度 〜 数段階の高速。空腹度減少・成長・孵化などの時間経過ロジックすべてに倍率がかかる)。観察・デバッグ用途

## キー操作(暫定)
```
f       餌やりを行う
m       薬を投げる(病気の治療)
p       一時停止 / 再開
[ / ]   シミュレーション速度を下げる / 上げる
R       水槽グレートリセット(確認あり)
+ / -   魚を1匹追加/間引く(デバッグ・任意)
,       設定画面(将来: 配色・fps 上限等)
?       ヘルプ
q       終了
```

## 状態の永続化
- 保存先: `~/.config/aquaterm/state.json`
- 内容: 各魚の種類・成長段階・空腹度・病気状態・座標、卵・餌・薬の状態など
- 起動時に読み込み、終了時(q)に保存。termmap の `--resume` 相当の「続きから」体験

## MVP の完了条件
1. 水槽背景(ブルー系グラデーション)+ステータスバーが表示される
2. 魚が複数種、なめらかに泳ぐ(f64 座標+diff 描画)
3. `f` で餌やりができ、魚が寄ってきて食べる
4. 空腹度(見た目反映込み)・成長・産卵→孵化・死亡が機能し、個体数が増減する
5. 病気の発症・進行・`m` での治療が機能する
6. `p` 一時停止、`[` `]` 速度変更、`R` グレートリセットが機能する
7. 終了時に状態を保存し、次回起動時に復元される

## MVP に含めないもの(将来検討)
- サウンド
- 複数水槽・レイアウト編集
- 水質(pH・水温など)の育成パラメータ(病気は含めるが、水質シミュレーションはしない)
- 設定のTOML化(termmap の config.toml 相当。最初は固定値でよい)
