# aquaterm 仕様書 (draft v1)

## 概要
aquazone 風の熱帯魚育成ツールをターミナルで動かす。termmap と同じく Rust + crossterm で自前レンダリングし、見た目のテイストを termmap に揃える(黒背景・下部反転ステータスバー・ハーフブロックによる高密度カラー描画)。配色はブルー系。

## 技術スタック
- Rust (edition 2021)
- crossterm: 端末描画・キー入力
- unicode-width: 全角文字幅計算(ステータスバー等)
- serde / serde_json: 状態の保存・読み込み
- ratatui は使わない(termmap と同じく自前 diff 描画。half-block で実質2倍解像度を出すため)

## 画面構成
- 上部: 水槽エリア(画面幅いっぱい、高さ = 端末高さ - 1)
- 下部: ステータスバー(反転表示)。餌の残数・魚の数・経過時間・キー案内を表示
- 起動時オンボーディング: termmap 同様、初回のみ簡易ヘルプを表示(以後出さない)

## カラースキーム(ブルー系)
- 背景(水): 上部が明るいシアン〜中段が青〜下部が濃紺のグラデーション(3〜4段階)
- 水底: 暗めのネイビー+砂の差し色(薄い黄土色を少量)
- 気泡: 白〜水色の点、上に向かってゆっくり移動
- 魚: 種ごとに差し色(オレンジ/イエロー/白など)を水槽の青との対比で目立たせる
- ステータスバー: termmap と同じ反転(明るい背景+黒文字)

## 描画方式
- 内部フレームバッファは「セル高さの2倍」の解像度を持つ論理ピクセルグリッド(上下2ピクセル分を上半分/下半分の色として1文字の半角ブロック `▀` に合成、termmap の half-block レンダラと同じ考え方)
- 前フレームとの diff のみ crossterm に書き込み、ちらつきを抑える
- tick は 30〜60fps 目標(可変、端末/描画コストに応じてフレーム間引き)。魚の座標は f64 で保持し、描画時にセル座標へ変換 → 補間により「ぬるぬる」動く

## 魚の挙動
- 座標・速度ベクトル(f64)を保持し、緩やかなランダムウォーク+慣性で自然に泳ぐ
- 水槽の壁で反射(端で急停止せず、手前で緩やかに向きを変える)
- 同種同士は緩く群れる(近傍平均方向に少し寄せる程度の軽い boid。過剰な実装はしない)
- 空腹度が低い個体は餌の位置に引き寄せられる(通常の遊泳ベクトルに餌方向への吸引ベクトルを加算)

## 魚種(dot-matrix デザイン、複数種)
- ネオンテトラ風の小型青系、金魚風のオレンジ、グッピー風の白+差し色の3種に加えて、**サメ型の大型種**を4種目として用意する(サメ=ヒレが尖った三角形、体が細長い、既存3種よりかなり大きいドット絵)
- 各種でサイズ・泳ぐ速さ・餌への反応速度が少し異なる
- サメ型の大型種は、既存3種と同じく空腹度・成長(稚魚→成魚)・病気・死亡の育成ロジックにフル参加する(「観賞用の追加生物」からは外し、通常の魚として扱う。個体数上限・`-`キー間引きの対象も通常魚と同じ扱いにする)
- **サメの入手経路はマニュアル(`S`キー)のみに限定(重要・方針転換)**: サメは自動的には増えない/出現しないようにする
  - 初期配置(`seed_initial`)・グレートリセット(`R`)ではサメを含めない(通常3種のみ)
  - `+`キー(ランダム追加)は**通常3種のみ**からランダム選択し、サメは絶対に選ばれないようにする
  - サメは産卵→孵化の繁殖ロジックからも除外する(サメの成魚は卵を産まない。サメを増やす唯一の方法は`S`キー)
  - 水槽にサメを入れる方法は `S`キー(専用ショートカット)のみ

## サメの捕食(新規)
- サメ型の魚は**捕食者**として、近くにいる他の魚(ネオン/金魚/グッピー。サメ同士は対象外)を検知し、近づいて襲う
- 頻度は控えめにする(四六時中は狙わない。例: サメの空腹度がある程度下がったときだけ捕食行動を取る、かつ1匹あたりクールダウン時間を設けて連続捕食を防ぐ)
- **追跡と回避の動き(方針変更: 常に逃げる)**: サメが空腹気味で捕食行動に入っているときは、対象の魚へ向かう吸引ベクトルを強めにかけて「追いかけている」のがわかる動きにする。通常の魚側は、**サメが空腹かどうかに関わらず、近くにサメがいるのを検知したら常に逃走ベクトルを加えて逃げる**(「みんなサメが嫌い」という設定。捕食モードかどうかの条件は撤廃)。**逃走はキビキビとした機敏な動きにする**: 通常時の速度よりはっきり速く、反応も鈍くならないようにする(腹ペコ時の餌への吸引と同様、通常の遊泳ロジックより優先されるくらいの強さでよい)
- **サメは追跡中は通常魚より速い**: 常時ではなく、**サメが追跡(捕食モード)に入っているときだけ**、その速度が既存3種の通常魚よりはっきり速くなるようにする(通常の巡回中は特別早くしなくてよい)。魚が機敏に逃げても、追跡中のサメの方が速いので追いつかれることがある、という緊張感を持たせる
- **回避を単純な直線逃走から「回り込み」に変える(見た目の改善)**: 現状の逃走ベクトルはサメから真っ直ぐ離れるだけの単純な動きになりがち。もう少し生き物らしく見えるよう、サメの進行方向に対して横(垂直方向)にずれる成分を混ぜ、ジグザグに切り返しながら回り込むような回避動作にする
- **ランダムな瞬発ダッシュ(新規・見た目のリアリズム向上)**: サメや餌など特定のトリガーがない通常時でも、魚が低頻度・ランダムなタイミングで一瞬だけ通常より速く素早く動く「ダッシュ」を行うようにする。方向もランダムでよい。生き物らしい躍動感を出すための演出で、頻度は控えめにする(頻発すると落ち着きがなく見えるため)
- 捕食されると対象の魚はその場で消える。**演出**: 捕食の瞬間、赤い血飛沫のような粒子エフェクトを散らす(既存の死亡演出=仰向けで浮いてゆっくり消える、とは別の専用演出にする)
- **捕食演出をもっと派手・グロテスクに強化(具体化)**: 「内臓が飛び散り、血が周囲に染み渡る」イメージで以下の2要素を組み合わせる
  - **破片パーティクル**: 単色の血飛沫ではなく、内臓を思わせる複数色(暗赤・赤黒・ピンク寄りの赤など)の破片を、範囲を広げて多めに飛び散らせる
  - **血の滲み(範囲エフェクト・再修正・さらに強め)**: 実装を確認したところ、固定半径(7px程度)のまま薄い赤み(混合率最大60%程度)がフェードアウトするだけで、「広がっていく」動きがなく地味だった。**同心円状に半径が時間とともに拡大していく波紋アニメーション**にしてほしい: 発生直後は小さく、時間経過で半径が大きく広がっていき(目安: 最大半径は現行の2〜3倍程度まで)、広がりながら徐々に薄くフェードアウトして消える。**色は数秒間「まっか」に見えるくらい濃くする**(発生直後〜数秒はほぼ赤一色に近いレベルで混合率を高くし、その後にフェードアウトする。今回のトーンは控えめではなく、はっきりインパクトのある赤にする)
  - **破片パーティクルも「内臓がどろーんと」する重量感を出す**: 前回依頼した複数色の破片パーティクルを、小さく軽い粒ではなく、もう少し大きく・重たげに見えるサイズ/動き(ゆっくり広がって沈んでいくような)にする
- サメの空腹度は捕食で大きく回復する(既存の餌でも回復はするが、捕食の方が効率よく回復してよい)
- ステータスバー等に捕食が起きたことの一言メッセージを出す(例: 「〇〇が食べられた…」)
- **専用ショートカット**: 新規キー `S`(大文字)でサメを1匹、水槽に確実に投入する。既存の `+`キー(ランダムな種類を追加)とは別に、狙ってサメだけを投入できるようにする。`ADD_FISH_MANUAL_CAP`(50匹)の対象に含め、上限に達していれば`+`と同様に追加しない(「これ以上は〜」の一言メッセージ)
- **サメは捕食するたびに大きくなる(新規)**: サメは魚を1匹捕食するごとに、見た目のサイズが段階的に大きくなっていく(捕食回数に応じたサイズボーナス)。上限を設けて無限に大きくならないようにする(例: 3〜5段階程度で打ち止め)

## 成長のさらなる段階(全種共通、新規)
- 既存の成長は稚魚(Fry)→成魚(Adult)の2段階だったが、**成魚になった後も満腹状態を維持し続けるとさらに段階的にサイズが大きくなっていく**ようにする(全種共通。サメは上記の捕食由来の成長と併用してよい)
- 段階には上限を設け、無限に大きくなり続けないようにする(画面を占有しすぎる・パフォーマンスに影響しない範囲。目安3〜5段階程度)
- 大きくなるほど泳ぐ速度がやや遅くなる、などの見た目・挙動の変化をつけてもよい(必須ではない)

## 育成要素
- **空腹度**: 時間経過で減少。餌を食べると回復。0 が一定時間続くと「弱っている」状態になる(死亡に至る条件・演出は「死亡についてv2」節を参照)
- **空腹度の減衰速度を緩和(実機フィードバック)**: 現行`HUNGER_DECAY`(満タン→0まで約10分、腹ぺこ閾値到達は約5分)は「腹ペコになるのが早すぎる」との指摘があった。目安として**満タンから0になるまで約60分(1時間)程度**まで減衰を緩やかにする(腹ぺこ閾値到達は概ね30分程度になる計算)。これに伴い死亡までの猶予(`STARVE_WEAK_TIME`/`STARVE_DEATH_TIME`)や病気関連の時間感覚は変えなくてよいが、実機で全体のバランス(空腹→弱り→死亡までのトータル時間)がおかしくなっていないか確認すること
- **空腹度の見た目反映**: 空腹度を3段階(満腹/普通/腹ぺこ)に分け、魚の挙動と色に反映する
  - 満腹: 泳ぐ速度が遅くゆったり、色は通常色のまま
  - 普通: 標準の速度・色
  - 腹ぺこ: 泳ぐ速度が上がり餌への吸引ベクトルが強まる、色をやや薄く/暗くして「弱っている」印象を出す。**実機フィードバック**: 現状の吸引効果は弱く、腹ペコでも餌にあまり群がって見えなかった。腹ペコ時は通常の遊泳ベクトルよりも餌への吸引ベクトルがはっきり優先されるくらい強めに効かせ、「腹ペコの魚が餌にめっちゃ寄ってくる」のが見た目でわかるようにする(近くに餌があれば一直線に向かうくらいの強さでよい)
- **ステータスオーバーレイ(腹ペコ/病気フラグ・生命ゲージ、ON/OFF切替可)**: 各魚のスプライト直上に以下をまとめて表示する。実機で確認したところ常時点灯のドットは水槽の背景色に埋もれて見えづらかったため、**点滅表示**にする(例: 0.5秒程度の間隔で点灯/消灯を繰り返す)
  - **腹ペコフラグ**: 空腹度が「腹ぺこ」閾値以下のとき、小さいマーク(記号または特定色のドット)を表示
  - **病気フラグ**: 病気の個体に、腹ペコフラグとは別の色/記号のマークを表示
  - **生命残りゲージ**: 元気度(空腹度と病気の深刻度から算出、死亡演出に近づくほど減っていく)を小さいバー(数セグメント)で表示。元気度が高いほど緑〜黄、低いほど赤に近づく
  - これらは`v`キーでまとめてON/OFFを切り替えられるようにする(既定は表示ON)。OFFにすると魚のスプライトと元気メーター無しのシンプルな見た目に戻る
- **成長**: 満腹状態が続くと稚魚→成魚へ段階的に成長(見た目サイズが変わる)
- **繁殖(産卵→孵化の2段階)**:
  - 成魚が満腹状態を一定時間維持すると、水底や水草付近に卵(2〜4個、小さいドット)を産む
  - 卵は空腹度を消費せず、孵化までの時間(一定時間)が経過すると孵化して稚魚になる
  - 孵化予定数が水槽の個体数上限を超える分は孵化しない(卵は消える)
  - **産卵頻度の調整(実機フィードバック)**: 現行`BREED_CHANCE_PER_SEC`(0.06/秒)は頻発しすぎ(「ポコポコ」生まれる)との指摘があった。産卵はちゃんと機能させつつ、目安として数十分に1回程度のイベントになるくらいまで確率を下げること(病気の頻度調整と同様、厳密な数式より実機での体感を優先して調整する)
- **個体数上限**: 端末サイズに応じた上限を設定する。大きい端末では最大100匹程度まで許容する(小さい端末ではそれに応じて低い上限になってよい)。100匹規模でも描画・シミュレーションが重くならないことを確認する
- **密度を上げてよい(緩和)**: 現行の `capacity(pix_w, pix_h) = (pix_w * pix_h) / 1400` だと、フォントサイズや端末ウィンドウが小さいと`+`キーの上限(50)や最大100に届きにくい。同じ端末サイズでもっと多く収容できるよう、除数(1400)を下げて密度を上げてよい(目安: 半分程度=700。パフォーマンスに問題が出ない範囲で調整する)
- **`+`キー(デバッグ追加)の上限を50に制限**: `+`キーによる直接追加は50匹までとし、それ以上(50〜個体数上限100)は産卵→孵化を経由してのみ到達できるようにする。個体数上限(端末サイズ依存、最大100)自体は変えず、`+`キーの実行条件だけ「現在の総数が50未満のときのみ動作する」ように制限する

## カーソル操作(新規)
- 水槽内に小さなカーソル(照準・十字マーク等、魚や背景と被らない色)を表示する。初期位置は画面中央
- 矢印キー(←↑↓→)でカーソルを移動する(水槽の範囲内に制限)
- `f`(餌やり)・`m`(薬)は、画面中央上部固定ではなく**カーソルのX座標**から投下する(Y座標は水面付近から開始し、沈む演出は維持)。狙った位置の魚に餌・薬を届けやすくする
- **投入位置の見える化**: `f`/`m` を押した瞬間、投下位置(カーソルのX座標、水面付近)で一瞬だけ光る/波紋のような視覚効果を出す。餌と薬で色を変え、どちらをどこに投げたか一目でわかるようにする
- **ガラスを叩く(こんこん)**: 新規キー `t` でカーソル位置の「ガラスを叩く」動作をトリガーする。カーソル位置に軽い振動/波紋エフェクトを一瞬出し、近くにいる魚は驚いて一時的にその位置から急いで離れる(既存の遊泳ベクトルに短時間だけ逃走ベクトルを加算する)。SEがあれば「コンコン」という短い音も鳴らす
- **叩きすぎのペナルティ**: 短時間(例: 数十秒〜1分程度の窓)に何度も連打すると、ストレスとして扱い、その周辺にいた魚の病気発症確率に一時的なボーナス(通常より発症しやすくなる)を与える。稀にしか叩かない分には影響が出ないようにし、乱用したときだけ効くようにする
- **逃走コスト**: ガラスを叩かれて驚いて逃げた魚は、驚いて逃げるのにエネルギーを使うという想定で、空腹度を目に見えて多く消費する(例: 逃走がトリガーされた瞬間に空腹度を一定量減らす、または逃走中は空腹度減少速度を大幅に上げる)。サメから逃げる場合も同様の考え方で消費してよい

## 餌やり操作
- `f` キー: カーソルのX座標付近から餌粒を複数個(3〜5粒)投下
- 餌はゆっくり沈み、水底に着くと停留する(下記「水底の食べ残し」参照。着水後すぐには消えない)
- 魚が餌に近づくと自動で食べる(空腹度回復、餌は消滅)
- **捕食量の制限(バグ修正・必須)**: 実機で確認したところ、餌を大量に投入して密集させた状態で魚が1匹通りかかると、その周辺の餌が一度に全部消費されて消えてしまう(1tickで魚1匹が何十粒も食べてしまう)。**1匹の魚が1tickで消費できる餌は1粒までに制限する**こと(最も近い1粒だけ食べて残りはそのまま残す=うまくスルーする)。薬(治療)も同様に、1匹の魚が1tickで消費できる薬は1つまでに制限する

## 水底の食べ残し・カニの掃除役(新規)
- 水底(砂の上)に沈んだ餌・薬は、これまでのように短時間で消えるのではなく、**水底に停留してしばらく溜まる**ようにする(水中を漂っている間の寿命ロジックはそのまま。水底着地後は寿命が止まる、または非常にゆっくりにする)
- **カニ**は歩いている最中に水底の餌・薬に近づくと、それを食べて消す(掃除役)。カニ自身の空腹度等は今回は実装しなくてよい(単に餌・薬を消費して片付けるだけの役割)
- **掃除スピードが速すぎる(バグ修正)**: 実機で確認したところ、カニが山になった食べ残しに近づくと範囲内を一括で全部消費してしまい、あっという間に山がなくなる。これは以前修正した魚の過剰消費バグと同種の問題(`update_crabs`が範囲内の着地物を`retain`で一括除去している)。**1匹のカニが1tickで消費できる餌・薬は1つまでに制限する**こと(魚側の1tick1粒制限と同じ考え方)。山を片付けるのに時間がかかる、ゆっくりした掃除になるようにする
- 水底に食べ残しが無限に溜まり続けて処理が重くならないよう、上限(例: 水底に留まれる餌・薬の総数に上限を設け、超えたら古いものから消す)を設けてよい
- **積もる見た目(山になる)**: 同じあたりに着地した餌・薬は、単に散らばって置かれるのではなく、少しずつ積み上がって「山」のように見えるようにする(例: 近い着地位置ならY座標を少しずつ上げて積み重ねる、一定の高さで裾野が広がるようにする等)。堆積している感じが視覚的にわかるようにする

## 病気・治療
- **発症**: 腹ぺこ状態が一定時間続く、または個体数が上限に近い過密状態のとき、確率で発症。発症すると見た目がまだら/くすんだ色になり、泳ぐ速度が鈍る。成長・繁殖は病気の間停止する
- **発症確率の緩和(実機フィードバック・目標値を再修正)**: 「すぐ病気になりすぎる」との指摘が続き、目標感を再設定する。**通常速度(x1)で丸1日(24時間)つけっぱなしにしていても、水槽全体で発症するのは数匹(3〜5匹)程度**というレベルまで、大幅に頻度を下げること。前回の目安「4倍速で10分に1回」では全く足りない(1日換算すると数百回発症してしまう桁違いの頻度になる)。`DISEASE_CHANCE_PER_SEC` は現行0.03/秒よりさらに1〜2桁小さい値まで下げる必要がある可能性が高い。`OVERCROWD_RATIO` も0.95程度に上げてよい。実際に24時間待つのは非現実的なので、最高速度(4倍速)で数時間放置する、または発症確率×発症条件下に留まる推定時間から逆算するなどして、目標(1日で水槽全体3〜5匹程度)に近づくよう調整し、実機で検証してから報告すること
- **放置した場合**: 空腹度0の衰弱と同様、放置すると「弱っている」状態が続き、さらに放置すると死亡に至る(「死亡についてv2」節の猶予・演出に従う)
- **治療**: `m` キーでカーソルのX座標付近から薬(餌と別色の粒)を投下。ゆっくり沈み、病気の魚が触れると治癒して通常状態に戻る。健康な魚が触れても何も起きない(誤投下してもマイナス影響なし)
- ステータスバーに「病気: N匹」を表示

## 観賞用の追加生物(方針転換・カニのみ)
- 大型魚(サメ型)は上の「魚種」節に統合し、通常の育成対象にした(この節からは除外)
- **カニ**: 水底(砂の上)を左右にゆっくり歩くエンティティ。泳がず水底ラインに固定され、時々立ち止まる。育成ロジック(空腹度・病気・死亡)には参加しない。観賞用の背景生物として残す
- カニは個体数上限・保存(state.json)の対象に含めてよいが、既存の魚とは別の軽量な構造体で管理してよい
- **`-`キー(間引き)の対象**: 通常の魚(サメ型含む)がまだいればそれを1匹間引く(既存の挙動)。通常の魚が0匹になったらカニを1匹間引く。`+`キーによる追加は通常魚(サメ型含む)のみでよい

## 死亡について(方針転換 v2・最終)
- MVP初版では空腹度0からわずか約85秒で餓死が始まり、数分で水槽が全滅する体験になったため、いったん「死なない」方針にした
- 最終的には**死亡は復活させるが、死ぬまでの猶予を大幅に延ばし、かつ死んだことが視覚的にわかる演出をつける**形にする
- **猶予**: 餌を一切与えなくても、空腹度が0になってから死亡に至るまでの合計時間が最低でも10分以上になるようにする。目安のパラメータ例(そのまま採用しなくてよいが、この程度の余裕を確保する):
  - 空腹度が満タンから0になるまで: 約10分程度(`HUNGER_DECAY` を現行の1.6/秒から大幅に下げる。例: 100/600秒 ≒ 0.17/秒)
  - 0になってから「弱っている」に入るまで: 約2分(`STARVE_WEAK_TIME` を8秒→120秒程度に)
  - 弱ってから死亡に至るまで: 約3分(`STARVE_DEATH_TIME` を22秒→180秒程度に)
  - 病気(`SICK_WEAK_TIME` / `SICK_DEATH_TIME`)も同様に大幅に延長する(現行12秒/30秒→60秒/180秒程度)。発症条件自体(`HUNGRY_SICK_TIME` / `DISEASE_CHANCE_PER_SEC`)は変えない
- **死亡演出**: 死亡判定に達した魚は、仰向け(逆さ)のスプライトに変わり、ゆっくり水面近くまで浮上して静止する(動きが止まる)。死亡の瞬間にステータスバーへ「○○が力尽きた…」等のメッセージを表示する
- 浮いた状態を数秒〜数十秒(目安10〜20秒程度)維持したのち、水槽から消える。即座に消えない
- 個体数上限に達したまま弱った/死んで浮いている魚が居座って繁殖が進まなくならないよう、繁殖(産卵)判定への影響がないか確認する

## 寿命・世代交代(新規)
- 各個体は生まれてからの経過時間(年齢)を記録する
- **老齢での産卵は確率アップでなく確定イベントにする(修正済み)**: 「老いると産卵確率が上がる」は生物学的に不自然なので撤回。代わりに、一定の年齢(老齢)に達したタイミングで**確定で1回だけ産卵**してから寿命死に向かう、という一度きりのイベントにする(満腹状態などの条件は問わず、老齢に達したら確定発生でよい)
- **寿命死**: さらに年を取ると、老衰で寿命死する。既存の死亡演出(仰向けで浮いてゆっくり消える)と同じ流れを使ってよい
- 対象は全種(サメも含む。サメが無限に居座り続けないようにする)
- 具体的な年齢の目安は指定しないので、これまで調整してきた時間感覚(空腹度は約1時間、病気/産卵は控えめな頻度)に合うよう、実機で確認しながら調整する。目安として、寿命は数時間〜半日程度のイメージで、頻繁に世代交代が起きて忙しくなりすぎない程度にする

## 水槽リセット・時間制御
- **グレートリセット**: `R`(大文字)キーで水槽を初期状態に戻す。誤操作防止のためステータスバーに確認プロンプトを出し、明示的な追加キー入力があった場合のみ実行する。魚を初期構成に戻し、卵・餌・薬・経過時間をすべて消去する
- **一時停止/再開**: `p` キーでシミュレーション全体(魚の動き・空腹度減少・成長・繁殖判定など)を停止/再開する
- **速度倍率**: `[` / `]` キーでシミュレーション速度を段階的に変更する(例: 停止相当の極小速度 〜 通常速度 〜 数段階の高速。空腹度減少・成長・孵化などの時間経過ロジックすべてに倍率がかかる)。観察・デバッグ用途

## ヘルプ画面の図鑑(新規)
- 現行のヘルプ画面は「種類: ネオン(青) / 金魚(オレンジ) / グッピー(白+差し色)」のようにテキストのみで種類を列挙している。これを、各魚種の**実際のドット絵スプライトも並べて表示する図鑑**のようなセクションに拡張する
- 対象は通常3種+サメ。名前・簡単な特徴(1行程度)・実際の色付きスプライト(成魚の見た目でよい)を並べる
- 端末が狭い場合はテキストのみにフォールバックしてよい(既存のロゴ幅ガードと同様の考え方)

## 効果音(SE、新規)
- 既存の音源ファイルは使わず、コード側で正弦波(+簡単な音量エンベロープ)を合成する短いビープ音にする(著作権フリー・依存を軽くするため)
- 対象イベント: 気泡が上る音(ぶくぶく。頻度は控えめに)、餌を入れた音、薬を入れた音、病気になった音、治療で回復した音、空腹(腹ぺこ)になった瞬間の音
- `s` キーでSEのON/OFFをトグルする(既定はON)
- オーディオ出力デバイスが無い/初期化に失敗する環境でもクラッシュせず、SEなしで通常通り動作すること(エラーは握りつぶしてよい)
- **音質(修正必須)**: 実機で確認したところ単一の定常サイン波が「笛のような単調な音」に聞こえ、安っぽかった。単一周波数を鳴らしっぱなしにするのではなく、**短時間の周波数スイープ(ピッチが時間で変化する)+速い減衰(ディケイ)**にすること
  - 餌: 高め→低めへ短時間(50〜80ms程度)で下降するピッチスイープ+速い減衰(「ぽちゃん」という水滴の質感)
  - 薬: 餌よりやや低め・やや長めのピッチダウンスイープ
  - 気泡: 低め→高めへの短いピッチアップスイープ、音量小さめ
  - 病気: 2音のわずかに不協和な音程で、ゆっくり下降
  - 回復: 明るい2〜3音の短いアルペジオ(上昇)
  - 空腹警告: 短く低めの一発音
  - ガラスを叩く(`t`): 短くパーカッシブな(立ち上がりが速く、すぐ減衰する)硬い音

## 自動餌やり・自動投薬・自動ガラス叩き(新規)
- 新規キー `a` で「自動モード」をON/OFFトグルする(既定はOFF)
- ONの間: 水槽に腹ペコの魚がいて、かつ漂っている(未着地の)餌が少ない場合、一定間隔で自動的に餌やり(`f`相当)を行う。病気の魚がいて、かつ漂っている薬が少ない場合、一定間隔で自動的に投薬(`m`相当)を行う
- 投下位置はカーソル位置にこだわらず、ランダムまたは画面中央でよい(自動なので狙わなくてよい)
- 頻繁に投下しすぎて餌・薬が氾濫しないよう、一定間隔(例: 数十秒に1回程度のクールダウン)を設ける
- **自動ガラス叩きも同じON/OFFに含める**: 自動モードON中は、ランダムな位置・タイミングで自動的にガラスを叩く(`t`相当)動作も時々発生させる。頻度は低めにする(数分に1回程度)。既存の「叩きすぎペナルティ」の判定にも自動こんこんはカウントされてよいが、頻度が低いので基本的にペナルティには引っかからない想定
- ステータスバー等に自動モードのON/OFF状態が分かる表示を出す

## 設定画面(`,`キー、これまでプレースホルダーだったものをちゃんと作る)
- `,`キーを押すと、既存の各種トグル項目を一覧して切り替えられる画面を開く(termmapの設定画面と同じ考え方: 実行中に一覧から選んで切り替え)
- 対象項目(既存キーと機能を共有してよい。設定画面はそれらへの分かりやすい入口を追加するもの):
  - 効果音(SE) ON/OFF(`s`と同じ状態を共有)
  - ステータスオーバーレイ ON/OFF(`v`と同じ状態を共有)
  - 自動モード ON/OFF(`a`と同じ状態を共有)
- 操作: 上下キーで項目を選択、Enterまたはスペースでトグル、Escで画面を閉じる
- 各項目の現在の状態(ON/OFF)を一覧上で分かるように表示する
- 端末が狭い場合の表示崩れ・パニックがないようにする(既存のロゴ幅ガード等と同様の考え方)

## キー操作(暫定)
```
←↑↓→   カーソル(照準)を移動
f       カーソル位置から餌やりを行う
m       カーソル位置から薬を投げる(病気の治療)
p       一時停止 / 再開
[ / ]   シミュレーション速度を下げる / 上げる
R       水槽グレートリセット(確認あり)
v       ステータスオーバーレイ(腹ペコ/病気フラグ・生命ゲージ)の表示ON/OFF
s       効果音(SE)のON/OFF
S       サメを1匹投入(専用ショートカット)
a       自動餌やり/自動投薬モードのON/OFF
+ / -   魚を1匹追加/間引く(デバッグ・任意)
,       設定画面(SE/オーバーレイ/自動モードを一覧して切替)
?       ヘルプ
q       終了
```

## 状態の永続化
- 保存先: `~/.config/aquaterm/state.json`
- 内容: 各魚の種類・成長段階・空腹度・病気状態・座標、卵・餌・薬の状態など
- 起動時に読み込み、終了時(q)に保存。termmap の `--resume` 相当の「続きから」体験

## MVP の完了条件
1. 水槽背景(ブルー系グラデーション)+ステータスバーが表示される
2. 魚が複数種、なめらかに泳ぐ(f64 座標+diff 描画)
3. `f` で餌やりができ、魚が寄ってきて食べる
4. 空腹度(見た目反映込み・ステータスオーバーレイ表示)・成長・産卵→孵化が機能し、個体数が増減する(死亡は「死亡についてv2」節の猶予・演出に従う)
5. 病気の発症・進行・`m` での治療が機能する
6. `p` 一時停止、`[` `]` 速度変更、`R` グレートリセットが機能する
7. 終了時に状態を保存し、次回起動時に復元される

## HTTP API(新規・別タスクで着手)
- ローカルループバック(127.0.0.1、既定ポート例: 7887)で軽量HTTPサーバーを立てる(`tiny_http` 等の軽量クレートを想定)
- エンドポイント案: `GET /status`(魚数・病気数・空腹度平均・経過時間等をJSONで返す, read-only)/ `POST /feed` / `POST /medicate`(既存のfeed/medicate相当の操作をトリガー)
- TUI側のSimulation状態と `Arc<Mutex<Simulation>>` 等で共有する
- 外部公開は想定しない(ループバックのみ)。認証は無しでよい(個人用途)
- HTTPサーバーの起動に失敗(ポート使用中等)してもTUI自体はクラッシュせず起動できること
- **設定ファイル化**: `~/.config/aquaterm/config.toml` に `[http] enabled = false` / `[http] port = 7887` を持たせ、起動時に読み込む。ファイルが無ければ `enabled = false`(既定オフ)で起動する。他の設定項目(配色等)は今回はconfig化しない・HTTP関連のみのスコープにする(「MVPに含めないもの」の設定TOML化は今回は見送りのまま)
- 現在進行中の一連の修正(死亡演出・カーソル・SE・ロゴ等)の完了後に着手する別タスク扱い
